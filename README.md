## Comparación entre Programación Tradicional y Programación Orientada a Objetos en Python

Estudiante: Andrango Nieto Alison Dayana

# Descripción del taller práctico

El presente taller práctico tuvo la finalidad de aplicar la programación tradicional y la programación orientada a objetos, utilizando ambas metodologías para representar un sistema de gestión de mascotas, cada una con sus respectivos elementos de programación.

# Taller estructurado en Programación Tradicional

Se implementó una solución estructurada basada en funciones, variables y entrada de datos por teclado, utilizando los fundamentos de la programación tradicional. Para ello, se desarrolló un sistema de descripción de mascotas en el que se registran datos como la especie, el nombre y la edad mediante una función. Posteriormente, la información ingresada por el usuario es mostrada en pantalla, lo que permite observar el funcionamiento de la programación tradicional.

# Taller estructurado en Programación Orientada a Objetos

Se implementó una estructura basada en programación orientada a objetos con la finalidad de aplicar el uso de clases y objetos. Para ello, se trabajó con dos archivos: mascota.py y main.py. En el archivo main.py se definieron las cuatro mascotas utilizadas en el sistema. Por su parte, en mascota.py se establecieron los atributos de especie, nombre y edad, además de los métodos encargados de mostrar la información de cada mascota y simular la emisión de un sonido.

# Reflexión

A través de este taller fue posible conocer de mejor manera la estructura básica de la programación tradicional y la programación orientada a objetos, observando que ambas permiten resolver un mismo problema, pero mediante enfoques diferentes. La programación tradicional se basa principalmente en funciones y variables, por lo que resulta adecuada para programas sencillos y de menor complejidad, como el sistema de registro de mascotas desarrollado en esta actividad.

Por otro lado, la programación orientada a objetos permitió trabajar con clases, objetos y métodos, logrando una organización más estructurada del código. Aunque su implementación requirió una estructura más extensa que la programación tradicional, permitió una mejor organización del programa.

En conclusión, ambas metodologías son útiles y cumplen con diferentes necesidades. La elección de una u otra dependerá del tipo de programa que se desee desarrollar y del nivel de complejidad que requiera el proyecto.
